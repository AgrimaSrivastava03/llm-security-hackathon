# Data Protection Impact Assessment (Starter)

- Purpose: security testing of LLM applications and runtime guardrails.
- Data types: may process PII in prompts/responses; avoid storing sensitive data unless necessary.
- Lawful basis: legitimate interest / contractual performance (customer security posture).
- Risks: accidental PII exposure; unauthorized access; over‑collection.
- Mitigations: redaction by default, retention limits, encryption, RBAC, audit logs, DSR workflows.
